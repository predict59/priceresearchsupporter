# 가격조사 PWA 간단 사용 설명서

이 문서는 국군복지단 가격조사 현장 입력 PWA의 기본 사용 흐름과 주요 기능을 정리한 간단 설명서입니다. 앱은 서버 DB를 사용하지 않고, 입력 자료와 현장 조사 데이터를 현재 기기의 브라우저 저장공간에 보관합니다. 다른 기기에서 이어서 작업하려면 반드시 백업 파일을 내려받아 복원해야 합니다.

## 1. 시작 전 준비

- 조사표 엑셀 파일과 마트 연락처 엑셀 파일을 준비합니다.
- 현장에서 사용할 기기에서 앱을 열고, 필요하면 홈 화면에 설치합니다.
- 최초 1회 업로드 후에는 지역, 마트, 품목 작업환경이 기기 안에 저장됩니다.
- 기기 변경, 브라우저 변경, 브라우저 데이터 삭제 시 기존 자료가 보이지 않을 수 있으므로 작업 중에는 지역별 백업을 자주 내려받습니다.

## 2. 전체 업무 흐름

1. **조사자료 업로드**
   조사표와 마트 연락처 엑셀을 올리면 앱이 지역, 마트, 품목 목록을 자동으로 구성합니다.

2. **지역리스트**
   조사할 지역을 선택합니다. 지역 카드에서 작업, 엑셀, 사진 ZIP, 백업을 실행할 수 있습니다.

3. **마트 리스트**
   마트명, 주소, 품목명, 품목코드, 바코드로 마트를 검색합니다. 필요하면 필터와 정렬을 열어 조사 상태별로 확인합니다.

4. **마트정보**
   마트 전경사진을 촬영 또는 선택하고, 담당자 연락처와 방문 조사일을 확인합니다. 마트사진이 없으면 조사 입력으로 넘어갈 수 없습니다.

5. **물품리스트**
   해당 마트의 조사 품목을 확인합니다. 품목명, 바코드, 품목코드, 담당자로 검색할 수 있으며 입력 버튼으로 가격정보 화면에 들어갑니다.

6. **가격정보 입력**
   실물 확인, 상태, 사진자료, 가격, 할인정보, 특이사항을 입력한 뒤 저장합니다. 저장 후 다음 미등록 품목으로 이동하거나 마트 리스트로 돌아갈 수 있습니다.

7. **지역별 내보내기**
   지역리스트에서 지역별 엑셀, 사진 ZIP, 백업 JSON을 내려받습니다.

## 3. 화면별 주요 기능

### 조사자료 업로드

- 조사표 엑셀과 마트 연락처 엑셀을 입력합니다.
- 업로드한 자료는 현재 기기의 브라우저 저장공간에 저장됩니다.
- 자료가 이미 저장되어 있으면 앱 실행 시 바로 지역리스트로 이동합니다.

### 지역리스트

- 지역별 마트 진행률과 품목 진행률을 확인합니다.
- `작업` 버튼으로 마트 리스트에 들어갑니다.
- `엑셀`, `사진`, `백업` 버튼으로 지역 단위 자료를 내려받습니다.
- 최근 작업 지역은 검색창 아래에 작은 배지로 표시됩니다.

### 마트 리스트

- 기본 화면은 전체 마트를 보여줍니다.
- 마트명, 주소, 품목명, 품목코드, 바코드 검색을 지원합니다.
- 마트카드에서 담당자 정보를 바로 확인할 수 있습니다.
- 마트 상태는 영업 중, 폐업, 임시휴업으로 표시됩니다.
- 방문 순서 편집 기능으로 현장 방문 순서를 직접 조정할 수 있습니다.

### 마트정보

- 마트 전경사진은 방문지당 1장만 저장됩니다.
- 사진 ZIP 내보내기 시 마트사진은 품목별 `.1` 사진으로 복사 생성됩니다.
- 폐업 처리 또는 임시휴업 처리를 선택하면 해당 마트의 하위 품목이 일괄 변경됩니다.
- 일괄 처리 시 이미 입력한 가격정보가 있어도 데이터가 바뀌므로 처리 전 확인창을 확인해야 합니다.

### 물품리스트

- 품목코드, 품목명, 바코드, 기준가격, 조사가격을 확인합니다.
- 조사 완료 품목은 적격 또는 부적격 배지가 표시됩니다.
- 사진이 입력된 품목은 작은 미리보기로 확인할 수 있습니다.

### 가격정보

- `정상진열 O`이면 규격일치, 바코드일치, 제품진열사진, 제품정보사진을 입력합니다.
- `정상진열 X`이면 상태, 바코드 등록여부, 판매여부, POS 사진을 입력합니다.
- 제품정보사진 촬영 시 브라우저가 지원하면 바코드 자동 비교를 수행합니다.
- 정상가 입력 시 기준가격과 비교해 적격 여부를 확인할 수 있습니다.
- 할인 있음 선택 시 행사여부, 할인기간, 구두 확인, 기간구분을 입력합니다.
- 특이사항은 폐점, 품절, 1+1 행사, 임시휴업, 판매처 미협조, 재고 소진, 재입고 예정 문구를 빠르게 추가할 수 있습니다.

## 4. 사진 입력 기준

- 마트사진: 마트정보 화면에서 1장 촬영 또는 선택합니다.
- 정상진열 O: 제품진열사진, 제품정보사진을 입력합니다.
- 정상진열 X: POS 또는 영수증 사진을 입력합니다.
- 사진은 저장 시 자동으로 적정 크기로 리사이징되어 브라우저 저장공간에 보관됩니다.
- 사진 ZIP 파일명은 `품목코드.1`, `품목코드.2`, `품목코드.3`, `품목코드.4` 형식으로 생성됩니다.

## 5. 내보내기와 백업

- 엑셀: 선택한 지역의 조사 결과를 엑셀로 내려받습니다.
- 사진 ZIP: 선택한 지역의 사진을 품목코드 기준 파일명으로 내려받습니다.
- 백업 JSON: 지역별 조사 데이터와 사진을 백업합니다.
- 다른 기기에서 이어서 작업하려면 기존 기기에서 백업 JSON을 내려받고 새 기기에서 복원합니다.

## 6. 사용 시 주의사항

- 앱 데이터는 서버에 저장되지 않습니다.
- 브라우저 저장공간을 삭제하면 앱 안의 조사 데이터도 삭제될 수 있습니다.
- 현장 작업 중에는 지역별 백업을 자주 내려받는 것이 좋습니다.
- 브라우저와 기기 환경에 따라 카메라, 파일 선택, 바코드 자동인식 지원 여부가 다를 수 있습니다.
- 자체저장공간 메뉴에서 현재 사용량과 예상 한도를 확인할 수 있습니다.
