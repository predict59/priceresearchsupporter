# 가격조사 PWA 간단 사용 설명서

이 앱은 국군복지단 가격조사 현장에서 마트별 품목 가격, 실물 확인, 사진자료를 입력하고 지역별 결과 파일을 내려받기 위한 오프라인 우선 PWA입니다. 조사자료와 현장 입력값은 서버 DB가 아니라 현재 기기의 브라우저 저장공간에 보관됩니다. 다른 기기에서 이어서 작업하려면 백업 파일을 내려받아 새 기기에서 복원해야 합니다.

## 1. 기본 업무 흐름

1. 조사표 엑셀과 마트 연락처 엑셀을 업로드합니다.
2. 앱이 지역, 마트, 품목 목록을 생성합니다.
3. 지역리스트에서 조사할 지역을 선택합니다.
4. 마트리스트에서 방문할 마트를 선택합니다.
5. 마트정보에서 마트 전경사진과 방문 조사일을 입력합니다.
6. 물품리스트에서 품목을 선택해 가격정보를 입력합니다.
7. 저장 후 다음 품목을 입력하거나 마트리스트로 돌아갑니다.
8. 지역리스트에서 엑셀, 사진 ZIP, 백업 JSON을 내려받습니다.

## 2. 화면별 사용법

### 조사자료 업로드

조사표 엑셀과 마트 연락처 엑셀을 입력하는 초기 설정 화면입니다. 파일을 업로드하면 지역, 마트, 품목 작업환경이 현재 기기의 브라우저에 저장됩니다. 바코드 고화질 파일은 품목별 바코드 값 확인용으로만 참고합니다.

### 지역리스트

조사 지역을 선택하는 화면입니다. 지역별 마트 진행률과 품목 진행률을 확인할 수 있고, 지역 단위 엑셀, 사진 ZIP, 백업 JSON을 내려받을 수 있습니다.

### 마트리스트

선택한 지역의 마트를 확인하는 화면입니다. 마트명, 주소, 품목명, 품목코드, 바코드로 검색할 수 있습니다. 필터와 정렬을 열어 조사 상태를 확인하거나 방문 순서를 직접 조정할 수 있습니다.

### 마트정보

마트 전경사진, 현재상태, 담당자 연락처를 확인하는 화면입니다. 현재상태는 미확인, 영업 중, 폐업, 임시휴업 중 하나를 선택합니다. 마트 전경사진이 있어야 조사 입력으로 넘어갈 수 있습니다. 폐점 또는 임시휴업 처리를 하면 해당 마트의 하위 품목 상태가 일괄 반영되고, 품목별 제품 사진은 정합성을 위해 정리됩니다.

### 물품리스트

선택한 마트의 품목 목록을 보는 화면입니다. 품목코드, 품목명, 바코드, 기준가격, 조사가격, 적격/부적격 여부를 확인할 수 있습니다. 사진이 등록된 품목은 작은 미리보기가 표시됩니다.

### 가격정보

품목별 조사값을 입력하는 화면입니다. 담당자 정보, 국군복지단 제시정보, 실물 확인, 상태, 사진자료, 가격, 특이사항 순서로 입력합니다. 실물 확인의 진열여부는 조사표 내보내기에서 정상진열로 표기됩니다. 비고 칩은 다시 누르면 해당 문구만 제거되며, 직접 입력한 문장은 보존합니다.

### 검증

미완료 품목, 사진누락 품목, 정상진열 X 품목, 특이사항 입력 품목을 확인하는 화면입니다. 항목을 누르면 해당 품목의 가격정보 화면으로 이동합니다.

### 백업/복원

현재 브라우저에 저장된 조사 데이터와 사진을 JSON 파일로 내려받거나 복원하는 화면입니다. 전체 백업은 업로드로 생성된 모든 지역, 마트, 품목, 사진, 설정을 포함합니다. 전체 백업 복원은 현재 기기의 기존 데이터를 초기화하고 백업 파일 기준으로 작업환경 전체를 다시 구성합니다.

## 3. 주요 기능

### 1. 입력 엑셀/사진 파일 양식에 맞게 일괄 다운로드

지역별 조사 결과 엑셀과 사진 ZIP을 내려받을 수 있습니다. 엑셀은 국군복지단 가격조사표 구조에 맞춰 기준정보, 실물 확인, 가격 판단, 정상진열 아닐 시, 특이사항, 조사일, 마트정보, 업체정보 열로 생성됩니다. 사진 ZIP은 품목코드 기준으로 생성됩니다.

### 2. 제품진열사진에서 가격 자동 추출

현재 버전에서는 지원하지 않습니다. 가격표 OCR은 향후 기능으로 검토할 수 있습니다. 현재는 조사가격을 직접 입력하고, 입력 즉시 기준가격과 비교해 경고 또는 확인 문구를 표시합니다.

### 3. 제품정보사진에서 바코드 자동 추출

브라우저가 바코드 자동인식을 지원하는 경우, 제품정보사진 촬영 후 바코드를 읽어 조사표의 바코드와 비교합니다. 값이 다르면 바코드 불일치로 표시해 사용자가 바로 확인할 수 있습니다.

### 4. 오프라인 현장 입력

초기 자료 업로드 후에는 인터넷이 불안정해도 지역, 마트, 품목 목록 조회와 가격 입력, 사진 첨부, 저장을 계속할 수 있습니다. 단, 데이터는 해당 기기의 브라우저에 저장되므로 주기적인 백업이 필요합니다.

### 5. 적격/부적격 자동 체크

저장된 조사값을 기준으로 적격/부적격을 자동 표시합니다. 미판매, 미진열, 바코드 미등록, 폐점, 임시휴업, 저가판매 등은 부적격으로 판단합니다. 정상 판매이고 국군복지단 제시 기준가격이 조사 가격 이하이면 적격으로 봅니다.

### 6. 마트 폐점/임시휴업 처리

마트정보 화면에서 폐점 또는 임시휴업 처리를 할 수 있습니다. 처리하면 이미 입력한 가격정보가 있어도 해당 마트의 전 품목 데이터가 일괄 변경됩니다. 실행 전 확인창이 표시됩니다.

### 7. 마트/제품 사진 미리보기

마트 전경사진과 제품 사진은 등록 후 화면에서 작은 미리보기로 확인할 수 있습니다. 잘못 올린 사진은 지우고 다시 촬영하거나 선택할 수 있습니다.

### 8. 마트/제품 리스트 임의 순서 정렬

마트리스트에서 방문 순서를 직접 조정할 수 있습니다. 사무실에서 외근 동선을 미리 정한 뒤, 현장에서는 지정한 순서대로 조사할 수 있습니다.

## 4. 사진 파일 규칙

- `품목코드.1.jpg`: 마트 전경사진
- `품목코드.2.jpg`: 제품진열사진
- `품목코드.3.jpg`: 제품정보/후면/바코드사진
- `품목코드.4.jpg`: POS 또는 영수증사진

마트 전경사진은 마트당 1장만 저장하지만, 사진 ZIP 생성 시 각 품목코드의 `.1.jpg` 파일로 복사됩니다.

## 5. 주의사항

- 브라우저 저장공간을 삭제하면 조사 데이터와 사진도 삭제될 수 있습니다.
- 같은 GitHub Pages 주소로 접속해도 기기나 브라우저가 다르면 데이터가 자동 공유되지 않습니다.
- 조사 중에는 지역별 백업 JSON을 자주 내려받아 보관하는 것이 좋습니다.
- 카메라, 파일 선택, 바코드 자동인식은 기기와 브라우저 정책에 따라 다르게 동작할 수 있습니다.
