# 국군복지단 가격조사 PWA 개발 현황 문서

이 문서는 가격조사 PWA의 요구사항, 현재 구현 내용, 운영 규칙, 향후 개발 방향을 한곳에 모아두기 위한 기준 문서다.
앞으로 기능을 추가하거나 정책이 바뀌면 이 문서도 함께 갱신한다.

최종 갱신일: 2026-07-01

관련 문서:

- `docs/USER_GUIDE.md`: 현장 사용자용 간단 사용 설명서

## 1. 프로젝트 목적

국군복지단 26-D 가격조사 현장 업무를 모바일/태블릿/PC에서 오프라인 우선으로 수행하기 위한 PWA 앱이다.

기존 업무는 엑셀 조사표를 열어 품목을 찾고, 사진을 따로 찍은 뒤, 나중에 파일명을 맞추는 방식이라 현장 입력과 사후 정리가 번거롭다.
이 앱은 조사표와 연락처 엑셀을 업로드한 뒤 현장에서 마트사진, 품목 가격, 실물 확인, 사진자료를 바로 입력하고, 지역별 엑셀/사진 ZIP/백업 JSON을 내보내는 것을 목표로 한다.

## 2. 기술 스택

- React
- Vite
- TypeScript
- PWA
- IndexedDB
- SheetJS/xlsx
- JSZip
- lucide-react
- GitHub Pages 배포

서버 DB는 사용하지 않는다.
데이터는 브라우저 IndexedDB에 저장된다.

## 3. 핵심 데이터 저장 구조

IndexedDB object store:

- `surveyFiles`
- `regions`
- `stores`
- `items`
- `photos`
- `settings`

주요 타입:

- `SurveyStore`: 방문 마트 단위
- `SurveyItem`: 품목 단위 조사 데이터
- `SurveyPhoto`: 사진 Blob
- `Region`: 지역/담당부서 정보
- `AppSettings`: 현재 지역, 기본 조사일 등

모든 주요 데이터는 `region` 기준으로 분리 관리한다.

## 4. 화면 구조

현재 화면명:

- 업로드
- 지역리스트
- 마트리스트
- 마트정보
- 물품리스트
- 가격정보
- 검증
- 백업/복원

상단 바 규칙:

- `지역리스트`: 우측 보조 문구 없음
- `마트리스트`: `현재 지역: 지역명`
- `마트정보`: `현재 마트: 마트명`
- `물품리스트`: `현재 마트: 마트명`
- `가격정보`: `현재 마트: 마트명`
- `검증`: `현재 지역: 지역명`
- `백업/복원`: 보조 문구 없음

본문의 중복 큰 제목은 가능한 제거하고, 상단 바 제목을 화면 제목으로 사용한다.

브라우저 뒤로가기:

- 앱 내부에서 뒤로갈 화면이 있으면 브라우저 뒤로가기도 앱 내부 뒤로가기로 처리한다.
- 최상위 화면에서 첫 뒤로가기는 `한 번 더 뒤로가기를 누르면 종료됩니다.` 안내를 표시한다.
- 안내가 표시된 상태에서 다시 뒤로가기를 누르면 브라우저 기본 뒤로가기를 허용한다.
- 모바일 브라우저에서 첫 뒤로가기 이벤트가 바로 이탈되지 않도록 히스토리 가드를 재확인한다.

## 5. 현재 구현된 주요 기능

### 5.1 엑셀 업로드/분석

- 조사표 엑셀 업로드
- 마트 연락처 엑셀 업로드
- 바코드 포함 조사표 업로드 칸은 존재하나 핵심 파싱은 기본 조사표 기준
- 최초 업로드 화면은 `자료 업로드` 화면으로 표시한다.
- 엑셀 파일 기반으로 작업환경이 생성되며 데이터가 서버 DB가 아니라 해당 기기의 브라우저 저장공간에 보관된다는 안내를 제공한다.
- 기기 전환 시 자동 연동되지 않으므로 백업/복원을 사용해야 한다는 안내를 제공한다.
- SheetJS로 엑셀 파싱
- 컬럼명 유연 매칭
- 조사표와 연락처 파일 병합
- 지역/담당부서 자동 구성

컬럼 유연 매칭 예:

- 제조사명, 회사명, 제조회사, 제조사
- 담당자연락처, 연락처, 전화번호, 업체연락처
- 마트번호, 마트 연락처, 판매처연락처, 매장번호
- 마트명, 판매처, 조사처, 매장명, 방문지
- 물품명, 품목명, 상품명, 제품명
- 기준가격, 기준가, 가격
- 주소, 마트주소, 판매처주소

### 5.2 지역리스트

- 지역명 검색
- 최근 지역 뱃지
- 지역별 담당부서 표시
- 지역 상세 범위 표시
- 마트 기준 진행률 카드
- 품목 기준 진행률 카드
- 지역별 엑셀 내보내기
- 지역별 사진 ZIP 내보내기
- 지역별 백업 JSON 내보내기

지역 구분은 제공된 기준표의 `지역` 단위로 한다.
예: 서울 1권역, 서울 2권역, 서울 3권역, 서울 4권역은 서로 다른 지역으로 유지한다.

### 5.3 마트리스트

- 기본 필터는 `전체`
- 완료 마트도 기본 화면에서 표시
- 마트명, 주소, 품목명, 바코드, 품목코드 검색
- 기본 정렬은 주소순
- 정렬 옵션:
  - 방문순서
  - 주소순
  - 미완료 많은 순
  - 품목 많은 순
  - 사진누락 많은 순
- 필터 패널은 접기/펼치기
- 마트 카드에는 마트사진 뱃지, 품목사진 누락 뱃지, 조사일, 진행률 표시
- 마트명 옆에 운영상태 뱃지 표시
- 긴 마트명은 카드에서 말줄임 처리하고 툴팁으로 풀네임 확인 가능
- 지도 링크는 마트 카드 메뉴에서 제공
- 담당자 정보는 버튼으로 모달 표시

검색어 관리:

- 지역 검색어, 마트 검색어, 물품 검색어는 각각 따로 관리한다.
- 마트리스트에서 검색 후 마트 안으로 들어가면 안쪽 검색어는 빈 상태다.
- 뒤로 마트리스트로 나오면 마트 검색어는 유지된다.
- 마트리스트에서 지역리스트로 더 올라가면 마트 검색어는 초기화된다.

### 5.4 방문순서 편집

마트리스트 필터 패널에서 `순서 편집`을 켜면 마트 카드 대신 한 줄 리스트가 표시된다.

표시 형식:

- 드래그 손잡이
- 방문순서 입력칸
- 마트명
- 주소
- 위/아래 버튼

기능:

- 번호 직접 입력
- 위/아래 버튼 이동
- PC에서 드래그 앤 드롭으로 순서 변경
- 변경된 방문순서는 `stores.visitOrder`에 저장
- 마트리스트 화면을 벗어나면 순서 편집 모드는 자동 해제

### 5.5 마트정보

- 마트명, 주소 표시
- 현재상태 표시: 영업 중, 폐업, 임시휴업
- 마트 전경사진 촬영/선택/삭제
- 마트 전경사진 미리보기 표시
- 마트 전경사진이 없으면 조사 입력 불가
- 마트 연락처 목록 표시
- 조사 품목 수 표시
- 방문 조사일은 마트정보 진입 시 오늘 날짜로 자동 선택
- 품목 저장 시 해당 마트의 방문 조사일을 품목 조사일로 저장
- 폐업 처리 버튼
- 임시휴업 처리 버튼
- 폐업/임시휴업 처리 시 확인 모달을 띄운 뒤 하위 모든 품목을 일괄 변경
- 일괄 변경 내용: 정상진열 X, 바코드등록여부 X, 판매여부 미판매, 가격 입력값 초기화, 특이사항에 `판매처 폐점` 또는 `임시휴업` 추가, 품목 상태 완료
- 조사 입력 버튼

마트사진은 방문지당 1장 저장한다.
사진 ZIP 내보내기 시 각 품목코드 `.1.jpg`로 복제된다.

### 5.6 물품리스트

- 품목명, 바코드, 품목코드, 담당자 검색
- 필터 접기/펼치기
- 품목 카드에 품목코드, 품목명, 바코드, 기준가격, 조사가격, 상태 표시
- 입력된 품목사진이 있으면 작은 미리보기 표시
- 조사가격이 입력된 경우 조사가격 옆에 `적격`/`부적격` 뱃지를 표시
- 적격/부적격 판단 기준:
  - 적격: 정상 판매 및 정상 가격, 즉 국군복지단 제시 기준가격 ≤ 조사 가격
  - 부적격: 미판매, 미진열, 바코드 미등록, 폐업/임시휴업, 저가판매
  - 31일 초과 할인은 할인가격을 판매가격으로 적용
  - 1+1 행사 문구가 있는 경우 입력된 판매가격을 2로 나누어 비교
- 품목 상태 뱃지 우측 정렬
- 품목 입력 버튼명은 `입력`

### 5.7 가격정보

입력 섹션:

1. 국군복지단 제시정보
2. 실물 확인
3. 상태
4. 사진자료
5. 가격
6. 특이사항

실물 확인:

- 정상진열 O/X가 핵심 분기
- 정상진열 O이면 규격일치/바코드일치 입력 가능
- 정상진열 X이면 규격일치/바코드일치 비활성화 및 공백 처리
- 정상진열 X이면 상태 입력 가능
- 정상진열, 규격일치, 바코드일치는 O/X만 사용한다.

상태:

- 바코드 등록 여부 O/X
- 판매여부: 미진열, 미판매
- POS 조회 여부: 조회함, 조회불가
- 섹션 제목 옆에 `(정상진열 X 시 입력)` 안내 표시
- POS 조회함 선택 시 비고에 `POS 확인` 문구를 자동 추가한다.
- POS 가격은 별도 입력하지 않고 정상가에 입력한다.

가격:

- 기준가격을 가격 입력 영역 상단에 표시
- 기준가격과 국군복지단 제시정보의 기준가격에는 `원` 단위를 표시
- 정상가 숫자 입력
- 가격 입력 중 천 단위 콤마를 표시한다.
- 정상가 입력 즉시 기준가격과 비교하여 아이콘 + 정상/주의 문구 표시
- 경고 문구가 여러 개면 줄을 나눠 표시한다.
- 기준가격과 30% 이상 차이 나면 추가 경고 표시
- 할인 없음/할인 있음
- 할인 없음이면 할인가, 할인기간, 구두확인, 날짜, 기간구분 비활성화
- 할인 있음이면 `행사여부`에서 1+1 행사 체크 가능
- `1+1 행사` 체크 시 정상가의 절반을 할인가로 자동 입력하고 비고에 `1+1 행사` 문구 추가
- 할인 있음이면 할인가 입력 후 할인기간을 `상시할인`, `기간 할인`, `모름` 중 선택
- 상시할인은 시작일/종료일 입력을 비활성화하고 기간구분을 32일 이상으로 자동 처리
- 기간 할인은 시작일~종료일 입력 시 31일 이내/32일 이상을 자동 계산
- 모름은 비고에 `할인 정보 확인 불가` 문구를 자동 추가
- 구두 확인 체크 시 비고에 `구두확인` 문구를 자동 추가
- 할인기간/구두확인/1+1 선택값이 해제되면 자동 추가된 비고 문구도 제거
- 엑셀 기간구분 열은 `1`, `2`, `1구두확인`, `2구두확인` 형태로 출력
- 정상진열 X이고 바코드 등록 여부가 X이면 가격 입력을 비활성화한다.

특이사항:

- 비정상진열 O/X
- 정상진열 X이면 비정상진열 입력은 비활성화한다.
- 비정상진열 O인 경우 엑셀의 비정상 컬럼에 O를 출력한다.
- 비정상진열 X인 경우 엑셀의 비정상 컬럼은 빈칸으로 둔다.
- 비정상진열 O이면 비고 작성 안내
- 자주 쓰는 문구 버튼: 폐점, 품절, 재고 소진, 재입고 예정, 1+1 행사, 임시휴업, 판매처 미협조
- 직접 비고 입력

저장 흐름:

- 저장 시 사진누락 확인
- 사진이 부족해도 앱 내부 확인 모달에서 사용자 확인 후 저장 가능
- 저장하면 상태는 완료
- 다음 미등록 품목이 있으면 이동 여부 확인
- 마트의 전 품목이 완료되면 마트리스트로 돌아갈지 확인
- 마트리스트로 돌아가면 해당 마트 카드가 포커스된다.
- 완료 마트가 기본 미완료 필터에 숨지 않도록 이 경우 자동으로 전체 필터로 전환한다.
- 하단 `목록`, `다음`은 저장하지 않고 이동한다.
- 저장되지 않은 변경사항이 있으면 앱 내부 확인 모달로 이동 여부를 묻는다.

## 6. 사진 규칙

### 6.1 사진 입력

마트사진:

- 마트정보 화면에서 촬영/선택
- 마트당 1장
- 업로드 후 미리보기 표시
- 사진 슬롯에는 별도 `업로드됨`/`미업로드` 문구를 표시하지 않고, 미리보기와 색상으로 상태를 구분한다.

정상진열 O:

- 제품진열사진
- 제품정보/후면/바코드사진
- 업로드 후 미리보기 표시
- 사진 슬롯에는 별도 `업로드됨`/`미업로드` 문구를 표시하지 않고, 미리보기와 색상으로 상태를 구분한다.
- 제품진열사진 설명: 가격정보와 진열상품이 동시노출 되도록 촬영
- 제품정보사진 설명: 상품후면 제품상세정보와 바코드 동시노출 되도록 촬영
- 정상진열 O 선택 시 바코드 자동 비교는 참고 안내로만 표시한다.
- 제품정보사진 업로드 후에는 `업로드 완료` 문구를 반복 표시하지 않고 바코드 인식 결과만 표시한다.

정상진열 X:

- POS/영수증사진
- 업로드 후 미리보기 표시
- 설명: 제품진열사진으로 가격정보 확인불가 시 POS기 또는 영수증 촬영

사진 누락 판단:

- 완료된 품목에 대해서만 품목사진 누락을 집계한다.
- 정상진열 O: 제품진열사진 또는 바코드/후면사진이 빠지면 누락
- 정상진열 X: POS/영수증사진이 빠지면 누락
- `판매처 폐점` 또는 `임시휴업` 일괄 처리 품목은 품목사진 누락 집계에서 제외한다.
- 마트사진은 별도 마트사진 뱃지로 관리

### 6.2 사진 리사이즈/압축

IndexedDB 용량 관리를 위해 사진 저장 전에 자동 리사이즈/압축한다.

현재 설정:

- 긴 변 최대 1280px
- JPEG 품질 0.72
- 원본보다 압축본이 작을 때만 압축본 저장
- 마트사진, 제품진열사진, 바코드/후면사진, POS사진 모두 적용

바코드 인식은 저장 전 원본 파일로 먼저 수행하고, 저장은 압축본으로 수행한다.

### 6.3 사진 ZIP 내보내기

사진 ZIP 파일명은 품목코드 기준이다.

- `품목코드.1.jpg`: 마트사진
- `품목코드.2.jpg`: 제품진열사진
- `품목코드.3.jpg`: 제품정보/후면/바코드사진
- `품목코드.4.jpg`: POS/영수증사진

마트사진은 마트당 1장만 저장되지만, ZIP 생성 시 각 품목코드 `.1.jpg`로 복사 생성한다.
사진이 없는 항목은 ZIP에 포함하지 않는다.

## 7. 바코드 자동 비교

제품정보/후면/바코드사진을 촬영 또는 선택하면 브라우저 내장 `BarcodeDetector`로 바코드 인식을 시도한다.

동작:

- 조사표 바코드와 일치하면 `바코드일치 O`
- 불일치하면 알림 표시 후 `바코드일치 X`
- 인식 실패 시 사진은 저장하고 실패 메시지 표시
- 브라우저가 지원하지 않으면 미지원 메시지 표시

인식률 개선:

- 전체 이미지 1회 인식
- 실패 시 2x2, 3x3, 상/하, 좌/우 영역으로 잘라 확대 인식
- 구석에 작게 찍힌 바코드도 인식 가능성을 높임

주의:

- 브라우저 지원 여부에 따라 동작이 달라질 수 있다.
- Chrome/Android 계열에서 가장 가능성이 높다.
- iOS/Safari는 지원이 제한적일 수 있다.

## 9. 백업/복원

지원 기능:

- 전체 백업 JSON 다운로드
- 백업 JSON 업로드
- 지역별 백업 JSON 다운로드
- 전체 데이터 초기화

백업 JSON에는 다음이 포함된다.

- stores
- items
- photos
- settings
- regions

사진은 Blob을 data URL로 변환하여 백업한다.

주의:

- 브라우저/앱/기기를 바꾸면 IndexedDB는 공유되지 않는다.
- 다른 폰에서 사용하려면 백업 JSON을 내려받고 새 기기에서 복원해야 한다.
- 앱 삭제, 브라우저 데이터 삭제, 사이트 데이터 삭제 시 IndexedDB도 삭제될 수 있다.

## 10. 저장공간/용량 운영 기준

이 앱은 브라우저 IndexedDB에 사진 Blob을 저장한다.
실제 저장공간은 기기 저장소를 사용하지만, 브라우저 정책과 기기 여유공간에 영향을 받는다.

현재 대응:

- 사진 자동 리사이즈/압축 적용
- 긴 변 1280px, JPEG 0.72
- 우측 상단 메뉴의 `자체저장공간`에서 브라우저 저장공간 사용량과 예상 한도를 확인 가능

권장 운영:

- 지역 단위 조사 완료 시 즉시 엑셀/사진 ZIP/백업 JSON 내보내기
- 백업 파일은 폰 내부에만 두지 말고 PC, 클라우드, 메신저, 메일 등으로 한번 더 보관
- 여러 날 작업하는 경우 매일 백업

향후 추가 권장 기능:

- 사진 개수/용량 표시
- 백업하지 않은 사진이 많을 때 경고
- `navigator.storage.persist()`로 영구 저장 요청

PWA 캐시:

- 기능이 바뀌었는데 반영이 안 되면 새로고침, 앱 완전 종료 후 재실행, 또는 사이트 데이터 삭제가 필요할 수 있다.
- 캐시 정책을 바꾸는 경우 `public/service-worker.js`의 `CACHE_NAME`을 올린다.

## 12. 현재 알려진 한계

- OCR 가격 자동 입력은 아직 미구현
- 바코드 자동 비교는 브라우저 지원 여부에 의존
- 드래그 방문순서 편집은 PC에서 더 안정적이며, 모바일에서는 위/아래 버튼을 보조로 사용
- IndexedDB는 서버 DB가 아니므로 기기 변경 시 자동 동기화되지 않음
- 사진이 매우 많을 경우 저장공간 관리와 주기적 백업이 필요
